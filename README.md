# OFFER-COMPARE

一个面向应届生的城市 Offer 决策工具。  
它不只比较“名义薪资”，而是比较你在真实生活里的 **购买力、时间价值、居住体感与长期职业收益**。

---

## 产品背景

### 应届生如何避免被“名义高薪”割韭菜？

很多候选人在拿到 Offer 时会优先看月薪或总包，但这会带来三个常见误判：

- **高薪幻觉**：一线城市薪资更高，但被房租和生活成本迅速吞噬。
- **时薪陷阱**：看起来涨薪，实际是更长工时换来的“用命换钱”。
- **长期价值盲区**：只看眼前现金，忽略履历背书和未来职业跃迁空间。

`OFFER-COMPARE` 的目标是把“感性纠结”变成“可解释的决策过程”，帮助应届生做出更稳健、更长期主义的职业选择。

---

## 决策模型（Decision Balance）

项目核心是一个可解释的幸福指数模型：

```text
HappinessIndex = (M * 0.4) + (T * 0.25) + (L * 0.2) + (C * 0.15)
```

- `M`：现金流安全（Money）
- `T`：时间自由（Time Value，真时薪）
- `L`：居住尊严（Livability，居住体感）
- `C`：职业跳板（Career Leverage，大厂背书）

### 为什么是 40 / 25 / 20 / 15？

- **40% 现金流安全**  
  对应届生来说，现金流安全感是决策底盘。可支配收入直接决定生活稳定性与抗风险能力，所以权重最高。

- **25% 时间自由**  
  只看总包会掩盖工时问题。引入真时薪后，可以识别“薪资上涨但单位时间价值下降”的情况，避免长期透支。

- **20% 居住尊严**  
  居住体感（可支配收入可租面积）是生活质量的重要代理指标，能直观反映城市成本差异对幸福感的影响。

- **15% 职业跳板**  
  大厂背书对未来跳槽、议价与简历信号有增益，但这是中长期收益，不应压过当下生活质量，因此作为补充权重。

> 这套权重不是“唯一真理”，而是一个面向应届生场景、可解释且可迭代的 PM 决策基线。

---

## 关键功能

- 城市生活成本对比（租房、餐饮、通勤、购物、水电网）
- 等价薪资换算（A 城市薪资在 B 城市需要多少才等价）
- 真时薪计算（基于工作制：955 / 996 / 大小周）
- 居住体感估算（基于房租比例换算可租面积）
- 决策天平可视化（四维权重进度条）
- 极端值警示（任一维度低于阈值时触发风险提醒）
- 本地状态持久化（上次输入自动回填）

---

## 技术栈

- **Vite**：快速构建与开发体验
- **Vanilla JS**：零框架依赖，逻辑透明、易读易维护
- **LocalStorage**：本地持久化，用户输入不离开浏览器

### 设计原则：轻量 + 隐私优先

- **轻量**：无需后端、无复杂依赖，打开即用，便于快速迭代和作品展示。
- **隐私优先**：所有输入只保存在本地浏览器，不上传个人薪资或职业偏好数据。
