# Offer Decision Copilot（OFFER-COMPARE）

一个面向应届生与跳槽者的 **AI PM 风格决策辅助系统 Demo**：  
不只比较「名义薪资」，而是把 **购买力、时间价值、居住体感、职业成长** 放进同一套 **可解释模型**，并用 **报告式解释（Explainability）** 与 **Trade-off 可视化** 呈现取舍关系。

它不是「黑盒打分器」，而是刻意强调：**推理路径可见、异常可降级、结论可质疑**。

---

## 为什么这不是普通计算器，而是 AI 决策辅助系统？

| 普通薪资计算器 | 本系统的 AI PM 取向 |
| --- | --- |
| 输出一个数字对比 | 输出 **多维效用 + 叙述性推理 + 可视化权衡** |
| 规则藏在脑子里 | **Explainability**：结论对应到「收入成本 / 时间 / 成长 / 幸福感 / 压力」主线 |
| 缺数据就报错或 NaN | **Fallback**：默认值、全国均值、规则模板、图表降级，保证可用 |
| 暗示存在「最优解」 | Trade-off 雷达强调：**没有完美 Offer，只有取舍** |

目标受众：**AI 产品经理 / 策略产品经理实习作品集** —— 展示 **产品抽象、风险控制、可解释 AI、数据可视化** 的综合能力。

## Explainability（可解释性）设计

### 用户输入 → 结构化报告

系统在结果页生成 **「AI 决策解释」** 模块（本地结构化叙事引擎，**非云端大模型**），输入包括：

- **城市**（决定基础生活成本结构）
- **月薪**（现金流底盘）
- **房租 / 通勤**（可选；缺失则 **Fallback** 到城市基准）
- **工作时长结构**（通过工作制 955 / 大小周 / 996 映射周工时）
- **成长空间自评**（受限 / 中等 / 充足 → 影响职业维度）

### 报告主线（至少覆盖）

1. **收入 vs 成本**：可支配现金流与等价换算口径一致。  
2. **时间自由**：真时薪视角，识别「名义高薪 + 隐性加班」风险。  
3. **长期成长**：成长自评 + 履历背书共同作用于职业维度。  
4. **幸福感**：四维加权幸福指数（与决策天平一致）。  
5. **压力水平**：合成「现金流紧张度、可支配缓冲、加班强度」的启发式指标，用于叙述而非单一分数审判。

### 叙述风格

- **自然语言、分段小标题、类分析报告**（非一条 if-else 结论）。  
- UI 采用 **卡片式 Copilot 解释面板**，强调可读性与复盘价值。

### 失败时的 Explainability

若结构化叙事生成异常，自动切换 **规则模板兜底**（见下文 Fallback），并在界面提示「已降级」，保证 **解释永不空白**。

## Trade-off（决策权衡）思维

### 四维雷达图

结果页提供 **Radar Chart**，对比两侧 Offer 在以下维度的 **相对强弱（0–100 归一化得分）**：

- **高收入**：现金流安全（与可支配收入相关）
- **时间自由**：真时薪 / 单位时间回报
- **居住舒适**：可租面积代理的居住体感
- **职业成长**：成长空间自评 + 履历背书

### 「没有完美 Offer」

雷达图刻意做成 **对抗式叠加（蓝 vs 紫）**：  
用户应一眼看出 **维度之间的拉扯**——例如「现金更高但时间更贵」或「更舒适但成长更慢」。

### 交互与可读性

- **图例**：城市 A / B 配色说明。  
- **指标说明**：脚注解释「得分为模型归一化，用于相对比较」。  
- **Hover Tooltip**：悬停顶点查看该维度释义与双方数值。

## Fallback（兜底）架构

所有兜底策略在代码中 **显式实现**（不仅写在 README），核心分布在：

- `src/fallbacks.js`：输入默认值、城市缺失时的 **全国均值**、房租/通勤缺失时的 **城市基准**  
- `src/explanation.js` + `src/explanationRules.js`：解释失败 → **规则模板**  
- `src/radarChart.js`：渲染或数据异常 → **「部分分析暂不可用」**，页面不崩溃  

### 行为一览

| 场景 | 行为 |
| --- | --- |
| 月薪缺失或非数字 | 使用默认月薪（见 `INPUT_DEFAULTS`） |
| 房租 / 通勤未填 | 使用城市库中的 **基准房租 / 基准通勤** |
| 城市在库中不存在（未来扩展自由输入时） | 使用 **全国聚合均值**（基于已加载 `cities.json` 计算） |
| 结构化解释抛错 | **规则模板** 输出 + Banner 提示 |
| 雷达数据 NaN 或渲染异常 | 清空图表区并提示 **部分分析暂不可用** |

## 决策模型（与 Trade-off / 解释共用）

加权幸福指数（与决策天平一致）：

```text
HappinessIndex = (M * 0.4) + (T * 0.25) + (L * 0.2) + (C * 0.15)
```

- **M** 现金流安全 · **T** 时间自由 · **L** 居住尊严 · **C** 职业跳板（含成长自评与背书）

权重是 **可解释的 PM 基线**，可在后续迭代中通过配置化或用户偏好学习升级。

## 关键功能清单

- 城市生活成本对比（租房、餐饮、通勤、购物、水电网）
- **可选自定义房租 / 通勤**（缺省走城市基准）
- **成长空间自评** 参与职业维度
- 等价薪资换算、真时薪、居住体感估算
- **AI 决策解释（结构化报告 + 规则兜底）**
- **Trade-off 雷达图（图例 + Tooltip）**
- 决策天平条形图、极端值警示
- 本地状态持久化（LocalStorage）

## 技术栈

- **Vite**：构建与开发体验  
- **Vanilla JS + ES Modules**：逻辑分层（`fallbacks` / `explanation` / `radarChart` / `main`）  
- **纯前端 SVG**：雷达图无第三方图表依赖，便于作品集阅读与部署  

### 设计原则

- **轻量**：无后端即可演示完整产品叙事。  
- **隐私优先**：薪资与偏好不上传服务器（默认）。  
- **可控 Demo**：强调 **风险控制与 Fallback**，契合 AI PM 实习作品叙事。
